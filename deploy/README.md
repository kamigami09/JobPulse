## How to get your LI_AT_COOKIE (for LinkedIn layer-2 scraping)

1. Open Chrome/Firefox and log into LinkedIn
2. Open DevTools → Application → Cookies → `https://www.linkedin.com`
3. Find the cookie named `li_at` and copy its Value
4. SSH to VPS and add it to `/var/www/jobpulse-backend/.env`:
   ```
   LI_AT_COOKIE=your_li_at_value_here
   ```
5. `systemctl restart jobpulse`

The `li_at` cookie expires when you log out of LinkedIn or after ~1 year.
Refresh it the same way if layer-2 scraping stops working.

## Updating the app

```bash
ssh root@187.127.229.82
cd /var/www/jobpulse-backend
git pull
venv/bin/pip install -r requirements.txt
FLASK_APP=run.py venv/bin/flask db upgrade
cd frontend && npm install && VITE_API_URL="" npm run build && cp -r dist/. /var/www/jobpulse/
systemctl restart jobpulse
```

## Useful commands

```bash
systemctl status jobpulse          # Service status
journalctl -u jobpulse -f          # Live logs
journalctl -u jobpulse --since today  # Today's logs
systemctl restart jobpulse         # Restart after config change
nginx -t && systemctl reload nginx # Test and reload Nginx
```
